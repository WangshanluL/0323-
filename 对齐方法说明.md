# 跨平台用户对齐系统 — 方法原理与融合策略

---

## 一、问题定义

**任务**：给定源平台 S 和目标平台 T 两个社交平台的用户聊天记录，对于源平台中每个待对齐用户，在目标平台中找出最可能是**同一自然人**的 top-K 候选账号。

**输入数据格式（每条消息一行）**：

| 字段 | 示例 | 说明 |
|------|------|------|
| account_id | u_001 | 用户唯一标识 |
| room_id | grp_12 | 所在群组 |
| msg | "今天加班好累" | 消息文本 |
| time | 1710489600 | Unix 时间戳 |
| type | text | 消息类型（text/image/video） |

**评估指标**：

| 指标 | 公式 | 直觉解释 |
|------|------|----------|
| **hit@1** | 排名第 1 就是正确答案的比例 | "一猜就中"的准确率 |
| **hit@K** | 正确答案出现在前 K 名中的比例 | 前 K 名的召回率 |
| **MRR** | 每个用户 1/排名 的平均值 | 综合反映排名质量，排越前分越高 |

> **例子**：有 4 个待对齐用户，正确答案分别排在第 1、3、未上榜、2 位。
> - hit@1 = 1/4 = 0.25（只有 1 人第 1 名就对了）
> - hit@10 = 3/4 = 0.75（3 人在前 10 名内）
> - MRR = (1/1 + 1/3 + 0 + 1/2) / 4 = 0.458

---

## 二、四种对齐方法

我们从四个完全不同的角度刻画"同一个人"的跨平台行为一致性，每种方法独立输出一份 top-K 排名：

```
              ┌─ GNN         社交结构 + 用词 + 作息 ──┐
              │                                       │
  源/目标消息 ─┼─ Style       作息频率 + 写作风格 ─────┤──→ 融合 ──→ 最终 top-K
              │                                       │
              ├─ UserPicture LLM 语义画像 ────────────┤
              │                                       │
              └─ Topic       主题分布 + 时序嵌入 ──────┘
```

### 2.1 GNN 方法 — 图神经网络结构对齐

#### 核心思路

> **"同一个人在不同平台，加入的群组类型相似、发言模式相似、活跃时段相似。"**

将用户和群组建模为**二部图**（用户 ↔ 群组，边权 = 发言条数），用图神经网络让每个用户的表示**融入其社群的集体信息**，再从多个维度计算相似度排名。

#### 特征体系（4 类，共 112 维）

| 特征 | 维度 | 数据形式 | 捕捉的信号 |
|------|------|----------|-----------|
| 文本 TF-IDF | 64 | 字符 2-4gram → TF-IDF → SVD 降维 | 个人用词习惯、口头禅 |
| 时间模式 | 31 | 24h 小时直方图(24维) + 星期直方图(7维) | 作息规律 |
| 群组指纹 | 11 | 群数、群规模均值/方差/最大值、发言占比等 | 社交圈结构 |
| 活动统计 | 6 | 消息数、活跃群数、消息长度、时间跨度（log 变换） | 行为量级 |

#### GNN 消息传播

在二部图上做 3 层传播，每层分两步：

```
① 用户 → 群组：群组节点 = 按发言比例聚合所有成员的向量
② 群组 → 用户：用户新向量 = 0.6 × 自身 + 0.4 × 所在群组的聚合
```

**传播的效果**：即使两个平台的群号完全不同，但如果一个用户在两边都参与"技术讨论群"（群内其他成员的词向量相似），GNN 传播后该用户的向量在两个平台上也会趋近。

#### 相似度打分

五路加权余弦相似度：

| 信号 | 权重 | 说明 |
|------|------|------|
| text_self（原始 TF-IDF） | 0.25 | 直接的用词相似 |
| text_gnn（GNN 传播后） | 0.30 | 结构增强的用词相似（最重要） |
| temporal（时间模式） | 0.15 | 作息是否吻合 |
| group_struct（群组指纹） | 0.20 | 社交结构是否相近 |
| activity（活动统计） | 0.10 | 行为量级是否匹配 |

#### 合理性举例

> 用户甲在**源平台**：参加 3 个技术群（群员 200-500 人），习惯晚 9-11 点发言，常用"其实""感觉"等词。
>
> 同一人在**目标平台**：也参加 3 个技术群（群号不同但规模相近），同时段活跃，同样的口头禅。
>
> → TF-IDF 捕捉到"其实""感觉"的匹配
> → 时间直方图在 21:00-23:00 区间高度重合
> → 群组指纹（3 个中等规模群、发言占比均匀）相似
> → GNN 传播让两边技术群成员的集体风格融入，进一步拉近向量距离
> → **综合得分远高于随机用户对**

### 2.2 Style 方法 — 频率 + 写作风格对齐

#### 核心思路

> **"同一个人有固定的作息节奏和独特的写作习惯——用短句还是长句、加不加标点、爱不爱发表情。"**

分两个子维度独立计算，再内部融合：

```
Style 方法
  ├─ 频率对齐（39 维）→ top-K 排名 ─┐
  │                                   ├─ Borda Count → Style 最终 top-K
  └─ 风格对齐（11 + TF-IDF 维）→ top-K 排名 ─┘
```

#### 频率特征（39 维）

| 特征 | 维度 | 数据形式 | 说明 |
|------|------|----------|------|
| 24h 发文直方图 | 24 | 每小时发言比例 | 经高斯平滑 + 幂次尖锐化处理 |
| 粗粒度时段 | 4 | 凌晨/上午/下午/晚间比例 | 弥补消息量少时 24h 直方图的噪声 |
| 峰值统计 | 4 | 圆统计均值、圆统计标准差、峰值占比、分布熵 | 精确定位活跃时刻 |
| 星期分布 | 7 | 周一到周日的发言比例 | 工作日 vs 周末 的习惯差异 |

**关于圆统计均值**：普通均值对跨 0 点的时间会算错（23:00 和 01:00 的均值不应是 12:00），圆统计将时间映射到单位圆上取向量均值，正确得出接近 0:00 的结果。

**关于尖锐化处理**：将直方图做 $p_i^\alpha$ 变换（$\alpha=1.5$），让高峰时段更突出、低谷更平坦，增强个人作息的辨识度。

#### 风格特征（11 + TF-IDF 维）

**非文本风格（11 维）**：

| 特征组 | 维度 | 具体指标 |
|--------|------|----------|
| 消息长度 | 4 | 均值、标准差、长消息比例（>50字）、短消息比例（≤10字） |
| 标点习惯 | 3 | 标点字符比例、句末标点比例、低标点标记（<5% 时为 1） |
| 媒体偏好 | 2 | 发图片占比、发视频占比 |
| 语气词 | 2 | 笑声词占比（哈哈/嘿嘿/lol）、感叹号占比 |

**文本 TF-IDF**（默认 2000 维）：字符级 3-5gram，捕捉个人惯用词组和口头禅。

> 两类特征以 0.3:3.0 的权重比拼接后计算余弦相似度（TF-IDF 更重要）。

#### 合理性举例

> 用户乙的习惯：**不加标点**、消息很短（均值 8 字）、喜欢发"哈哈哈"、主要周末晚上活跃。
>
> 频率特征：星期六/日在 20:00-24:00 有明显高峰
> 风格特征：short_ratio=0.80, low_punct_flag=1, laugh_ratio=0.25
>
> 另一个平台上的同一人保持了相同习惯 → 频率向量余弦相似度 > 0.9，风格向量余弦相似度 > 0.85
> 而随机用户之间这两个值通常在 0.2-0.4 → **区分度很高**

### 2.3 UserPicture 方法 — LLM 用户画像语义对齐

#### 核心思路

> **"同一个人在不同平台关注的领域、生活方式、说话风格是一致的，即使具体用词完全不同。"**

用大语言模型（LLM）阅读每个用户的全部发言，提取**结构化画像标签**，然后通过标签集合的相似度做匹配。这是唯一能捕捉**深层语义**的方法。

#### 处理流程

```
原始消息 ──→ 按用户聚合全部 text 消息 ──→ LLM 生成画像 JSON ──→ 画像相似度排名
```

#### 画像结构

LLM 为每个用户生成一个结构化 JSON：

```json
{
  "profile_summary": "关注编程与健身，表达风格轻松幽默",
  "topic_labels": {
    "兴趣话题": ["编程", "开源"],
    "关注内容": ["健身", "跑步"]
  },
  "lifestyle_labels": {
    "作息习惯": ["夜猫子"],
    "娱乐方式": ["手游"]
  },
  "language_style_labels": {
    "表达方式": ["口语化"],
    "情绪表达": ["幽默", "自嘲"]
  },
  "emotion_tone": "积极",
  "content_language": "中文",
  "confidence": "高"
}
```

每个标签字段按类别组织，每个类别最多 2 个标签，避免过度发散。

#### 相似度计算

将三类标签字段（topic + lifestyle + language_style）展平为集合后：

$$\text{sim}(u_s, u_t) = 0.70 \times \frac{|L_s \cap L_t|}{|L_s \cup L_t|} + 0.15 \times \mathbb{1}[\text{emotion 相同}] + 0.15 \times \mathbb{1}[\text{language 相同}]$$

- **0.70**：标签 Jaccard 相似度（核心信号）
- **0.15**：情绪基调是否一致
- **0.15**：主要语言是否一致

#### 合理性举例

> 用户丙在**源平台**用中文讨论："今天 leetcode 刷了两道 hard""推荐一个 Go 的 ORM 框架"
> 同一人在**目标平台**用英文说："just solved 2 hard problems on LC""anyone knows a good Go ORM?"
>
> GNN 的 TF-IDF 和 Style 的 TF-IDF 都**无法匹配**（语言不同，字符 n-gram 完全不重叠）。
>
> 但 LLM 能理解两边都在讨论**编程和算法**，生成相似的标签：
> - 源平台标签：`{编程, 算法, Go}`
> - 目标平台标签：`{programming, algorithm, Go}`
>
> → Jaccard 可能不高（中英文标签不同），但如果 LLM 统一用中文标签则可以直接匹配
> → emotion=积极 一致，language=中英混合 可能一致 → 总分显著高于随机用户对
>
> **核心价值**：补充其他三种方法在"跨语言/跨表述"场景下的盲区

### 2.4 Topic 方法 — 主题时序嵌入对齐

#### 核心思路

> **"同一个人在不同平台，会在相近的时间段讨论相同的话题。"**

用预训练模型将每条消息编码为语义向量，聚类出 80 个隐含话题，按天聚合每个用户的话题分布，通过**时间窗口匹配**找到跨平台的时序共现。

#### 处理流程

```
每条消息 ──→ BAAI/bge-m3 编码(1024维) ──→ KMeans 聚类(80个话题)
         ──→ 按天聚合：话题分布(80维) + 日均嵌入(1024维) + 全局嵌入(1024维)
         ──→ 时间窗口内余弦相似度 ──→ 加权融合 ──→ top-K
```

#### 特征体系

| 特征 | 维度 | 构建方式 | 捕捉的信号 |
|------|------|----------|-----------|
| 每日话题分布 | 80 | 当天各簇消息占比，L2 归一化 | "这天在聊什么" |
| 每日平均嵌入 | 1024 | 当天所有消息嵌入均值 | 更细粒度的语义 |
| 全局平均嵌入 | 1024 | 该用户全部消息嵌入均值 | 整体内容偏好 |

**每日话题分布示例**：

```
用户丁，3月15日发了 10 条消息：
  簇 0（健身）: 3条 → 30%
  簇 2（工作）: 5条 → 50%
  簇 7（娱乐）: 2条 → 20%
  → 话题向量: [0.30, 0, 0.50, 0, 0, 0, 0, 0.20, 0, ...]（80维）
```

#### 时间窗口匹配

核心思想：源平台第 D 天的话题只与目标平台第 D ~ D+2 天做比较（允许 2 天偏移）。

```
源平台用户丁，3月15日话题向量 vs 所有目标候选：
  候选 B 的 3月15日: 余弦相似度 0.97 ← 很像！
  候选 C 的 3月16日: 余弦相似度 0.21 ← 不像
  候选 D 的 3月16日: 余弦相似度 0.94 ← 也很像

→ 取 max = 0.97，配合其他天的分数，最终 B 排名最高
```

**为什么允许时间偏移？** 两个平台的活跃时段可能不同，某人白天在一个平台聊了某话题，可能次日才在另一个平台发相关内容。窗口容纳了这种合理延迟。

#### 三信号加权融合

| 信号 | 默认权重 | 含义 |
|------|---------|------|
| topic_score（话题分布匹配） | 0.70 | "在聊什么"的主题一致性 |
| embed_score（日级嵌入匹配） | 0.30 | 具体措辞的语义接近度 |
| global_score（全局嵌入匹配） | 0.10 | 长期内容偏好的一致性 |

权重归一化后使用。

#### 合理性举例

> 3月15日某热点事件爆发，用户丁在两个平台都大量讨论。
>
> → 两平台 3月15日的话题向量在"热点事件"簇上高度集中 → 余弦相似度 > 0.9
> → 随机用户在同一天恰好讨论同一话题的概率极低
> → 连续多天的话题共现构成了极强的身份指纹
>
> **核心价值**：利用"时间 × 话题"的二维共现模式，区分力远强于仅看话题或仅看时间

## 三、四种方法对比

| 维度 | GNN | Style | UserPicture | Topic |
|------|-----|-------|-------------|-------|
| **核心信号** | 社交结构 + 用词 + 作息 | 作息频率 + 写作风格 | LLM 提取的兴趣/风格标签 | 话题分布 + 时序嵌入 |
| **语义深度** | 浅层（字符 n-gram） | 浅层（字符 n-gram + 统计） | 深层（LLM 理解） | 中层（预训练嵌入） |
| **时间利用** | 隐式（直方图） | 显式（24h + 星期） | 无 | 显式（按天窗口匹配） |
| **结构利用** | 强（二部图 GNN） | 无 | 无 | 无 |
| **计算开销** | 低（纯 CPU） | 低（纯 CPU） | 中（LLM 推理） | 高（GPU 嵌入） |
| **擅长场景** | 群组结构明显 | 作息/写作习惯鲜明 | 跨语言/跨表述 | 事件驱动、时序规律 |
| **薄弱场景** | 无群组或群差异大 | 活动量少、无明显习惯 | 内容太少画像模糊 | 无共同事件、时区差大 |

**互补性分析**：

```
场景 A：用户在两个平台加入相同类型的群 → GNN 强，其他方法弱
场景 B：用户习惯凌晨 2 点发言、从不加标点  → Style 强
场景 C：用户两边用不同语言聊相同领域     → UserPicture 强，其他方法弱
场景 D：某周连续讨论同一热点事件         → Topic 强
场景 E：多个信号同时存在               → 融合后互相印证，准确率最高
```

四种方法在特征空间上几乎正交，融合后能覆盖绝大多数场景。

## 四、融合策略

### 4.1 当前基线 — Borda Count 等权投票

每种方法对每个 source 用户输出 top-K 排名，按排名位置赋分：

```
排名第 1 → 得 K 分，排名第 2 → 得 K-1 分，...，排名第 K → 得 1 分
```

四路得分相加，取总分最高的前 K 个候选。

**示例（K=5）**：

```
源用户 A 的四路排名：

GNN:         [X(1), Y(2), Z(3), W(4), V(5)]
Style:       [Y(1), X(2), Z(3), U(4), W(5)]
UserPicture: [Z(1), Y(2), X(3), W(4), U(5)]
Topic:       [Y(1), Z(2), X(3), W(4), V(5)]

Borda 得分（K=5）：
  用户Y: 4+5+4+5 = 18 ← 第1  ✓ 假设这是正确答案
  用户X: 5+4+3+3 = 15 ← 第2
  用户Z: 3+3+5+4 = 15 ← 第3（同分按字典序）
  用户W: 2+1+2+2 =  7 ← 第4
  用户V: 1+0+0+1 =  2 ← 第5
```

GNN 认为 X 最像（排第 1），但 Style、UserPicture、Topic 都更倾向 Y，**三票对一票，Y 胜出**。

**Borda Count 的局限**：

1. **等权**：不管某方法的置信度有多高或多低，得分公式都一样
2. **丢信息**：只用排名位置，丢弃了原始相似度分数
3. **无条件依赖**："GNN 排第 1 且 Topic 排前 3"比"四方都排第 5"更有说服力，但 Borda 无法表达

### 4.2 升级方案 — 基于决策树的学习融合

#### 基本思想

把每个 **(source 用户, 候选 target)** 对视为一个样本，从四种方法中提取特征，训练一个决策树模型来学习**"什么样的排名/分数组合代表真正的匹配"**。

#### 特征设计（12 维）

| 特征 | 来源 | 说明 |
|------|------|------|
| `gnn_rank` | GNN | 该候选在 GNN 结果中的排名（不在列表中设 K+1） |
| `gnn_score` | GNN | GNN 的加权余弦相似度原始分数 |
| `style_rank` | Style | 该候选在 Style 结果中的排名 |
| `style_score` | Style | Style 的余弦相似度原始分数 |
| `up_rank` | UserPicture | UserPicture 排名 |
| `up_score` | UserPicture | 画像 Jaccard 相似度分数 |
| `topic_rank` | Topic | Topic 排名 |
| `topic_score` | Topic | Topic 三信号融合分数 |
| `rank_agree` | 交叉 | 四方中排进 top-3 的方法数（0~4） |
| `best_rank` | 交叉 | 四方中的最优排名 |
| `avg_score` | 交叉 | 四方分数的均值 |
| `rank_std` | 交叉 | 四方排名的标准差（越小 = 越一致） |

标签：

- **正样本（y=1）**：该 (source, candidate) 在真值表中是正确匹配
- **负样本（y=0）**：不是正确匹配

#### 训练流程

```
┌──────────────────────────────────────────────────────────┐
│  Step 1：四种方法独立运行，输出 top-K 排名 + 原始分数     │
│                                                          │
│  GNN     → {user_s: [(cand, rank, score), ...]}          │
│  Style   → {user_s: [(cand, rank, score), ...]}          │
│  UP      → {user_s: [(cand, rank, score), ...]}          │
│  Topic   → {user_s: [(cand, rank, score), ...]}          │
└───────────────────────┬──────────────────────────────────┘
                        ↓
┌──────────────────────────────────────────────────────────┐
│  Step 2：构造候选池 = 四方 top-K 候选的并集               │
│                                                          │
│  对每个 source 用户：                                     │
│    candidates = union(GNN_topK, Style_topK,              │
│                       UP_topK, Topic_topK)               │
│    对每个 candidate 提取 12 维特征 + 标签                  │
└───────────────────────┬──────────────────────────────────┘
                        ↓
┌──────────────────────────────────────────────────────────┐
│  Step 3：训练决策树（按 source 用户分组交叉验证）          │
│                                                          │
│  model = GradientBoostingClassifier(                     │
│      max_depth=4, n_estimators=100,                      │
│      class_weight={0:1, 1:20}   ← 正负样本均衡           │
│  )                                                       │
│  model.fit(X_train, y_train)                             │
└───────────────────────┬──────────────────────────────────┘
                        ↓
┌──────────────────────────────────────────────────────────┐
│  Step 4：预测 & 重排序                                    │
│                                                          │
│  proba = model.predict_proba(X_test)[:, 1]               │
│  对每个 source 用户的候选池按 proba 降序 → 输出 top-K     │
└──────────────────────────────────────────────────────────┘
```

#### 决策树的可解释性

训练完成后可以直接读出树的分裂规则，**人能看懂模型在做什么**：

```
                    ┌─ rank_agree >= 3 ?
                    │
               ┌── YES ──→ best_rank <= 2 ?
               │            │
               │       ┌── YES ──→ 预测为正（置信度 0.94）
               │       │           "三方以上都排前3且有方法排前2 → 几乎确定是对的"
               │       │
               │       └── NO ──→ avg_score >= 0.5 ?
               │                    │
               │               ┌── YES ──→ 预测为正（0.82）
               │               └── NO  ──→ 预测为负（0.71）
               │
               └── NO（rank_agree < 3）
                    │
                    └── gnn_rank <= 1 AND topic_score >= 0.7 ?
                         │
                    ┌── YES ──→ 预测为正（0.78）
                    │           "虽然只有2方认可，但GNN排第1且Topic分数很高"
                    │
                    └── NO ──→ rank_std >= 5 ?
                         │
                    ┌── YES ──→ 预测为负（0.92）
                    │           "各方排名分歧很大 → 不太可能是对的"
                    │
                    └── NO ──→ 预测为负（0.65）
```

**核心优势**：决策树能学到**条件化的权重分配**——某种方法在特定条件下更可信时多听它的，在另一条件下不可信时降低权重。这是 Borda Count 做不到的。

#### 完整示例

```
源用户 A 的候选池（四方并集共 8 个候选）：

候选  GNN排名  GNN分  Style排名 Style分  UP排名  UP分  Topic排名 Topic分  agree best  std
 Y      2     0.71     1      0.85      2     0.72     1      0.88     4    1    0.50
 X      1     0.82     2      0.78      3     0.61     3      0.75     4    1    0.82
 Z      3     0.65     3      0.70      1     0.80     2      0.83     4    1    0.82
 W      4     0.58     5      0.45      4     0.55     4      0.61     0    4    0.41
 V      5     0.51    K+1     0.0      K+1    0.0      5      0.55     0    5    2.36
 ...

Borda Count（K=5）：
  Y: 4+5+4+5 = 18
  X: 5+4+3+3 = 15
  Z: 3+3+5+4 = 15  ← X 和 Z 并列，需要额外规则打破

决策树 predict_proba：
  Y: 0.94  ← rank_agree=4, best_rank=1, avg_score=0.79 → 走到最高置信叶子
  Z: 0.91  ← rank_agree=4, best_rank=1, up_score 最高
  X: 0.88  ← rank_agree=4, gnn_rank=1
  W: 0.22  ← rank_agree=0, std 低但分数也低
  V: 0.08  ← rank_agree=0, std 高, 信号弱

最终 top-3: [Y, Z, X]
```

决策树利用了 Borda Count 丢弃的**原始分数信息**和**一致性特征**，给出了更精准的排序。

### 4.3 对比总结

| 维度 | Borda Count（当前） | 决策树融合（升级） |
|------|---------------------|-------------------|
| 需要训练数据 | 不需要 | 需要真值表 |
| 利用的信息 | 仅排名位置 | 排名 + 分数 + 交叉特征 |
| 权重分配 | 固定等权 | 自适应、条件依赖 |
| 可解释性 | 高（规则透明） | 高（可输出分裂规则） |
| 应对方法失效 | 差（失效方法仍等权投票） | 好（学到何时忽略某方法） |
| 预期提升 | 基线 | 通常 +5~15% hit@1 |

**推荐路线**：先用 Borda Count 验证四种方法各自有效 → 确认有足够标注数据后 → 训练决策树融合模型进一步提升。
