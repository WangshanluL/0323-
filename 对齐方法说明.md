# 跨平台用户对齐方法说明

本项目实现了三种独立的用户对齐方法（GNN、Style、Topic），以及基于 Borda Count 的投票融合。本文档介绍每种方法的核心思路、所用指标，并配有示例说明。

---

## 背景：问题定义

**任务**：给定源平台（如微信群）和目标平台（如抖音）的用户消息记录，对于源平台中指定的一批用户，在目标平台中找出最可能是同一个人的账号，输出 top-K 候选列表。

**输入**：
- 源/目标平台的消息 CSV（字段：`account_id`, `room_id`, `msg`, `time`, `type`）
- 待对齐的 source 账号列表
- （可选）真值表，用于评估

**输出**：每个 source 用户对应的 top-K target 候选列表

**评估指标**：
| 指标 | 含义 | 计算方式 |
|------|------|----------|
| hit@1 | 排名第 1 的候选就是真实目标的比例 | 正确在第 1 位的用户数 / 总用户数 |
| hit@K | 真实目标出现在 top-K 中的比例 | 正确在前 K 位的用户数 / 总用户数 |
| MRR | 平均倒数排名，综合衡量排名质量 | Σ(1 / 真实目标排名) / 总用户数 |

> **举例**：50 个用户中，20 人的目标排在第 1 位，另有 10 人排在第 2~10 位。
> hit@1 = 20/50 = 0.40，hit@10 = 30/50 = 0.60，
> MRR = (20×1 + 10×0.5 + ...) / 50（考虑每人的具体排名）。

## 方法一：GNN（图神经网络特征融合）

> 文件：`GNN/alignv3.py`

### 核心思路

**"同一个人在不同平台的行为模式是相似的"**：包括说话内容的风格、活跃时段、参与的群组类型、发言活跃程度等。GNN 方法提取 5 类信号，并通过图神经网络让用户特征在群组结构中传播，增强表示能力，最后用加权余弦相似度排名。

### 第一步：构建二部图

将每个平台的数据建模为**用户-群组二部图**：用户节点和群组节点之间的边权重 = 该用户在该群组的发言条数。

```
用户 A ——(30条)—— 群组1
用户 A ——(5条) —— 群组2
用户 B ——(20条)—— 群组1
```

### 第二步：提取 5 类特征信号

| 信号 | 维度 | 权重 | 内容 |
|------|------|------|------|
| 文本 TF-IDF（原始）| 8000→64 | 0.25 | 字符级 n-gram（2~4字符），SVD 降维到 64 维 |
| 文本 TF-IDF（GNN传播后）| 64 | 0.30 | 用 GNN 在图上传播后的文本特征 |
| 时间分布 | 31 | 0.15 | 24小时发文直方图（24维）+ 星期分布（7维）|
| 群组结构 | 11 | 0.20 | 加入的群数、群规模均值/方差/最大值、发言占比等 |
| 活跃度统计 | 6 | 0.10 | 总消息数、活跃群数、平均消息长度、时间跨度、发帖频率等（均取 log） |

#### 文本 TF-IDF 特征

把每个用户的所有消息拼成一篇"文章"，用**字符级 n-gram TF-IDF** 编码（2~4个字符为一个 token）。这样能捕捉个人用词习惯，例如：

> 用户甲习惯说"哈哈哈"、"好的好的"，这些字符组合在他的向量中会有高权重。

提取后用 SVD 降维到 64 维，使跨平台的向量维度一致，便于相似度计算。

#### 群组结构特征（11维）

```python
feats[uid] = [
    log(加入的群数),           # 1维
    群规模均值, 群规模标准差, 群规模最大值,  # 3维
    发言占比均值, 发言占比标准差,           # 2维
    top5群的规模...                         # 5维
]
```

**示例**：源平台用户 A 加入了 3 个大群（各 500 人），发言比例均匀；目标平台用户 B 也加入了 3 个大群，发言模式相近 → 群组结构特征相似，得高分。

### 第三步：GNN 消息传递

核心思想：**同一个群里的成员会相互影响彼此的特征表示**。比如一个技术群里的用户，其 TF-IDF 词向量中应该有技术词汇的信息传入。

每一层 GNN 分两步：

**① 用户 → 群组聚合**（群组节点更新）：
```
h_群组[g] = Σ_用户 (该用户在g的发言比例 × h_用户)
```

**② 群组 → 用户聚合**（用户节点更新）：
```
h_用户_新 = 0.6 × h_用户_旧  +  0.4 × Σ_群组 (发言比例 × h_群组)
```

重复 3 层，每层末尾 L2 归一化。

**举例（第1层）**：

```
初始：用户A(文本向量=[1,0,0,...])，加入群组1（发言占比40%）

群组1节点 ← 用户A 40% + 用户B 30% + 用户C 30%（聚合所有成员）

用户A更新 ← 60% 自身 + 40% 群组1节点

→ 用户A的向量融入了群组的"集体风格"
```

这使得同一个话题群中的用户在两个平台上的向量更加相近，提高了跨平台匹配精度。

### 第四步：加权融合打分

对每个 source 用户，计算其与所有 target 用户的综合得分：

```
得分 = 0.25 × 原始TF-IDF余弦相似度
     + 0.30 × GNN传播后TF-IDF余弦相似度
     + 0.15 × 时间分布余弦相似度
     + 0.20 × 群组结构余弦相似度
     + 0.10 × 活跃度统计余弦相似度
```

按得分降序取 top-K 作为候选。

## 方法二：Style（频率 + 写作风格）

> 文件：`style/user_alignment.py`（融合），`style/align_frequency.py`，`style/align_style.py`

### 核心思路

**"同一个人在不同平台有相似的作息规律和写作习惯"**：一个人惯于晚上 10 点发消息，喜欢用短句，少用标点，爱发照片，这些习惯跨平台保持一致。Style 方法分别计算频率特征和风格特征，再用 Borda Count 投票融合。

### 频率特征（39 维）

#### 1. 24 小时发文直方图（24 维）

统计每个用户在一天 24 个小时中的发言比例：

| 小时 | 0时 | 1时 | … | 22时 | 23时 |
|------|-----|-----|---|------|------|
| 发言比例 | 0.01 | 0.00 | … | 0.15 | 0.08 |

> 额外处理：对直方图做**高斯平滑**（缓解数据稀疏的锯齿），再做**尖锐化**（α>1，让峰值更突出，使高活跃时段更加显著）。

**举例**：源平台用户 A 在 21:00~23:00 发言占 40%，目标平台用户 B 也在同一时段发言占 38% → 24h 直方图高度相似 → 高余弦相似度。

#### 2. 粗粒度时间段（4 维）

将一天分为 4 段，统计各段发言比例：

```
深夜/早晨 [0-6)   上午 [6-12)   下午 [12-18)   晚上 [18-24)
   0.05           0.10          0.35            0.50
```

适合消息量少、精细直方图噪声大的情况。

#### 3. 峰值时间特征（4 维）

| 特征 | 含义 | 示例值 |
|------|------|--------|
| 圆统计均值 | 高活跃时刻的"平均时间"（处理 0/24 跨界） | 0.92（约 22:00） |
| 圆统计标准差 | 活跃时间的集中程度 | 0.08（很规律） |
| 峰值占比 | 最高峰对应的发言比例 | 0.18 |
| 分布熵 | 发言是否均匀分散 | 0.72（较分散） |

> **圆统计均值的必要性**：若某人习惯在 23:30 发消息，普通均值会被 0:30 拉向 12:00，而圆统计均值会正确计算出接近 0 点的时刻。

#### 4. 星期分布（7 维）

统计周一到周日的发言比例，捕捉"工作日活跃"还是"周末活跃"的习惯。

### 风格特征（11 + TF-IDF 维）

#### 非文本风格（11 维）

**① 消息长度统计（4 维）**：
```
平均长度、长度标准差、长消息比例（>50字符）、短消息比例（≤10字符）
```

**举例**：
- 用户甲：平均消息 8 个字，短消息比例 80%（典型短回复型）
- 用户乙：平均消息 60 个字，长消息比例 70%（典型详细描述型）
- 这两人的长度向量差异很大 → 不可能是同一人

**② 标点使用习惯（3 维）**：
```
标点字符比例（，。！？等占总字符数）
句末标点比例（以。！？结尾的消息占比）
低标点标记（标点比例 < 5% 则为 1，否则为 0）
```

**举例**：某人说话习惯不加标点，"好的 明白了 下午见"；另一人很规范，"好的。明白了！下午见。" → 标点特征明显不同。

**③ 媒体类型比例（2 维）**：
```
发图片消息占比、发视频消息占比
```

**④ 语气词与感叹特征（2 维）**：
```
含"哈哈/嘿嘿/lol"的消息比例（笑声习惯）
含感叹号的消息比例
```

#### 文本 TF-IDF（500~8000 维）

用**字符级 n-gram TF-IDF**（2~5 字符）提取每个用户的写作风格向量。与 GNN 中的 TF-IDF 不同，这里更注重个人惯用词组、口头禅：

> 例如某人习惯用"就是说"、"那个那个"、"哈哈哈哈哈"，这些在字符 n-gram 向量中占有高权重。

两种聚合方式：
- `concat`：把所有消息拼接后一次向量化
- `mean`：对每条消息分别向量化后取均值

### 内部 Borda Count 融合

频率对齐和风格对齐各自产生一个 top-K 候选排名，内部用 Borda Count 投票融合：

```
Borda 得分：排名第 i 的候选得 K - i + 1 分（第1名得K分，第K名得1分）

source 用户 A 的候选：
  频率方法排名: [用户X(1), 用户Y(2), 用户Z(3)]
  风格方法排名: [用户Y(1), 用户X(2), 用户W(3)]

Borda 得分（K=3）:
  用户X: 3 + 2 = 5  ← 综合第1
  用户Y: 2 + 3 = 5  ← 综合第2（同分按字典序）
  用户Z: 1 + 0 = 1
  用户W: 0 + 1 = 1

最终: [用户X, 用户Y, 用户Z/W, ...]
```

## 方法三：Topic（主题分布 + 语义嵌入）

> 文件：`topic/topic_theme_match.py`

### 核心思路

**"同一个人在不同平台讨论的话题在时间上是对齐的"**：如果源平台用户在某天大量讨论了"健身"话题，那么真实对应的目标平台用户也应该在附近几天有类似的"健身"相关发言。Topic 方法用深度语义嵌入 + 主题聚类 + 时间窗口匹配来捕捉这种对应关系。

### 第一步：语义嵌入

使用预训练模型 **BAAI/bge-m3**（多语言、高质量 sentence transformer，向量维度约 1024）将每条消息编码为一个稠密向量。

```
消息："今天去健身房练了腿" → [0.12, -0.34, 0.78, ...]（1024维）
消息："leg day at the gym"  → [0.11, -0.33, 0.77, ...]（接近！）
```

跨语言语义相近的消息会映射到向量空间中相邻的位置，适合跨平台场景。

### 第二步：主题聚类

对所有消息的嵌入向量做 **K-means 聚类**（默认 80 个簇），每个簇代表一个"话题"：

```
簇 0：健身/运动相关消息
簇 1：美食/餐厅相关消息
簇 2：工作/加班相关消息
...
簇 79：...
```

对每条消息，根据最近的簇中心分配话题标签（硬分配）或按相似度分配概率分布（软分配）。

### 第三步：构建用户"指纹"

对每个用户的每一天，分别构建：

**① 每日话题分布向量（80 维）**

统计该用户当天各话题的消息比例，归一化后作为该天的"话题指纹"：

```
用户 A，2024年3月15日：
  簇0(健身): 3条消息 → 30%
  簇2(工作): 5条消息 → 50%
  簇7(娱乐): 2条消息 → 20%
  其余簇: 0
  → 话题向量: [0.30, 0, 0.50, 0, 0, 0, 0, 0.20, ...]（80维，归一化）
```

**② 每日平均嵌入向量（1024 维）**

将该用户当天所有消息的嵌入向量取均值，再 L2 归一化：

```
用户A，3月15日的 10 条消息嵌入取均值 → 代表当天的"语义中心"
```

**③ 全局平均嵌入向量（1024 维）**

将该用户所有消息的嵌入取均值，作为整体风格的全局特征。

### 第四步：时间窗口匹配打分

对于一对 source 用户 A 和 target 用户 B，用时间窗口方式计算相似度：

```
for 用户A的每一天 src_day:
    在目标侧找时间窗口 [src_day, src_day + 2天] 内的所有天
    计算 max(目标侧各天的指纹 与 src_day指纹 的余弦相似度)
取所有天的均值作为话题得分
```

**举例**：

```
源平台用户A，3月15日的话题向量: [0.30, 0, 0.50, ...]（健身+工作日）

目标平台候选们：
  用户B，3月15日: [0.28, 0, 0.52, ...]  → 余弦相似度 0.97（非常像！）
  用户C，3月16日: [0.10, 0.70, 0.15, ...]  → 余弦相似度 0.21（不像）
  用户D，3月17日: [0.32, 0, 0.48, ...]  → 余弦相似度 0.94（很像）

→ 3月15日这天: max(0.97, 0.21, 0.94) = 0.97 （匹配用户B）
```

**为什么用时间窗口而不是精确同天匹配？**
两个平台的时区、活跃时间段可能有差异，某人白天在微信群里聊健身，可能是在抖音上当天或次日才发相关内容。允许 ±2 天的窗口可以容纳这种延迟。

### 第五步：多信号融合

最终得分 = 三种信号的加权和：

```
总得分 = 0.7 × 话题分布相似度
        + 0.3 × 每日嵌入相似度
        + 0.1 × 全局嵌入相似度
```

- **话题分布（权重最高）**：最能反映"在谈什么"
- **每日嵌入**：更细粒度的语义，捕捉具体措辞
- **全局嵌入**：整体写作风格的稳定特征

## 三路融合：Borda Count 投票

> 文件：`align_ensemble.py`

### 核心思路

三种方法各有侧重：GNN 擅长利用群组社交关系，Style 擅长捕捉作息习惯和写作风格，Topic 擅长匹配语义话题。单一方法都有盲区（比如两人作息相同但无关联、或话题重合但不是同一人），三路投票可以互相补充，降低噪声。

### Borda Count 算法

每种方法对每个 source 用户输出一个 top-K 候选排名，得分规则：

```
排名第 1 的候选 → 得 K 分
排名第 2 的候选 → 得 K-1 分
...
排名第 K 的候选 → 得 1 分
排名 K 以后的候选 → 得 0 分
```

三种方法的得分相加，取总分最高的前 K 个候选。

### 完整示例（K=5）

```
源平台用户 A 的三路候选排名：

GNN方法:   [用户X(1), 用户Y(2), 用户Z(3), 用户W(4), 用户V(5)]
Style方法: [用户Y(1), 用户X(2), 用户Z(3), 用户U(4), 用户W(5)]
Topic方法: [用户Z(1), 用户Y(2), 用户X(3), 用户W(4), 用户V(5)]

Borda 计分（K=5）：
  用户X: 5 + 4 + 3 = 12 分  ← 第2名
  用户Y: 4 + 5 + 4 = 13 分  ← 第1名  ✓ 假设这是真实目标
  用户Z: 3 + 3 + 5 = 11 分  ← 第3名
  用户W: 2 + 1 + 2 =  5 分  ← 第4名
  用户V: 1 + 0 + 1 =  2 分  ← 第5名
  用户U: 0 + 2 + 0 =  2 分  ← 第5名（同分）

最终候选: [用户Y, 用户X, 用户Z, 用户W, 用户V]

→ 真实目标用户Y 排名第1 → hit@1 +1 ✓
```

本例中，GNN 认为用户X最像（排第1），但 Style 和 Topic 都更倾向用户Y，投票后 Y 胜出，修正了 GNN 的误判。

### 三种方法的互补性

| 场景 | GNN | Style | Topic |
|------|-----|-------|-------|
| 两人均加入同类型大群 | ✓ 强 | 弱 | 弱 |
| 两人作息时间完全相同 | 弱 | ✓ 强 | 弱 |
| 两人跨平台持续讨论同话题 | 弱 | 弱 | ✓ 强 |
| 三种信号均弱 | × | × | × |

三路投票的优势：只要其中两种方法正确识别出目标，投票结果大概率正确。

## 评估说明

| 指标 | 公式 | 典型值（本数据集）|
|------|------|-----------------|
| hit@1 | 正确排第1的用户数 / 总用户数 | GNN: 0.38 |
| hit@10 | 正确在前10的用户数 / 总用户数 | GNN: 0.88 |
| MRR | Σ(1/rank) / 总用户数 | GNN: 0.54 |

**MRR 计算示例**：

```
用户1：目标排名第1  → 1/1 = 1.000
用户2：目标排名第3  → 1/3 = 0.333
用户3：目标未在前K  → 0
用户4：目标排名第2  → 1/2 = 0.500

MRR = (1.000 + 0.333 + 0 + 0.500) / 4 = 0.458
```

MRR 对排名靠前更敏感：目标排第1贡献1.0，排第2贡献0.5，排第10贡献0.1，比 hit@K 能更细致地反映排名质量。
