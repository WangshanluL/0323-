这套对齐系统的核心思路是：同一个人在两个平台上，虽然账号ID不同，但"行为指纹"会高度相似。代码从5个维度提取指纹，加权融合后排序找最像的人。下面逐一举例说明。

---

## 整体框架

假设源平台有用户 Alice_src，要在目标平台几千人中找到她的对应账号 Alice_tgt。做法是给每个用户算5个维度的特征向量，然后对每个维度分别算 Alice_src 和所有目标用户的相似度，加权求和后取 top-K。

---

## 信号1：用户自身文本 TF-IDF（权重 0.25）

**原理**：同一个人在不同平台说话用词习惯相似——常用口头禅、表情符号、专业术语等。

**做法**：把每个用户的所有消息拼成一篇"文档"，用字符级 n-gram（2~4个字符）做 TF-IDF 向量化。字符级而非词级是因为聊天消息短、口语化，字符 n-gram 能捕捉打字习惯（比如"哈哈哈"vs"哈哈"、"emmm"等）。

**举例**：Alice 习惯说"确实"、"哈哈哈"、"666"，这些高频字符片段在她的 TF-IDF 向量中权重高。目标平台中同样高频使用这些词的人，余弦相似度就高。

---

## 信号2：GNN 聚合后的文本特征（权重 0.30，最高）

**原理**：光看个人用词还不够，同一个人会出现在"对应的群"里。GNN 通过群组中转，让用户"吸收"同群成员的特征，使得在对应群里的同一人特征进一步趋近。

**做法**：先把 TF-IDF 用 SVD 降到 64 维作为初始特征，然后做 3 轮消息传递：

```
第1阶段：群组特征 = 该群所有成员特征的加权平均（权重=成员在群内消息数占比）
第2阶段：用户特征 = 0.6×自身旧特征 + 0.4×所在各群特征的加权平均
```

每轮重复上述两步，迭代3次。

**举例**：假设"摄影群"在源平台和目标平台各有一个对应群。Alice_src 在源平台摄影群里和 Bob_src、Carol_src 一起聊天。经过 GNN 聚合后，Alice_src 的特征向量会融入 Bob 和 Carol 的用词特征。目标平台那边，Alice_tgt 也在对应摄影群里和类似的人聊天，聚合后特征自然趋近。这个信号权重最高（0.30），因为群组中转提供了很强的结构信息。

**加权细节**：如果 Alice 在摄影群发了100条消息、群总共500条，她对该群聚合的贡献权重就是 100/500=0.2。反过来，如果她在3个群，摄影群100条、工作群200条、闲聊群50条，那摄影群对她的回传权重是 100/350≈0.286。

---

## 信号3：时间行为模式（权重 0.15）

**原理**：同一个人在两个平台的作息时间相同——几点活跃、周几爱聊天。

**做法**：统计每个用户消息的"小时分布"（24维）和"星期分布"（7维），归一化成概率分布，拼成31维向量。

**举例**：Alice 是夜猫子，晚上10点~凌晨1点消息最密集，周末比工作日活跃。她的时间向量在 hour[22]、hour[23]、hour[0] 和 dow[5]、dow[6] 位置值高。目标平台中作息一致的人相似度就高。

---

## 信号4：群组结构指纹（权重 0.20）

**原理**：同一个人加入群的"模式"相似——加了多少群、群大小分布、在各群的活跃占比等。

**做法**：为每个用户提取11维特征向量：

| 维度 | 含义 |
|------|------|
| log(群数量) | 社交广度 |
| 群大小均值 | 偏好大群还是小群 |
| 群大小标准差 | 群规模的多样性 |
| 最大群大小 | 是否参与大型群 |
| 消息占比均值 | 平均活跃程度 |
| 消息占比标准差 | 活跃度是否集中 |
| top5群大小 | 主要群的规模排列 |

**举例**：Alice 加了5个群，其中1个200人大群（她发言占1%）、4个10人小群（她发言占15%~20%）。这个"少量大群+多个小群、小群更活跃"的模式就是她的结构指纹。目标平台的对应账号大概率也是类似模式。

---

## 信号5：活跃度统计（权重 0.10）

**原理**：同一个人发消息的数量级、频率、消息长度习惯在两个平台大致一致。

**做法**：6维特征，全部取 log(1+x) 压缩尺度：

| 维度 | 含义 |
|------|------|
| log(消息总数) | 总量级 |
| log(群数量) | 参与广度 |
| log(平均消息长度) | 是发长文还是短句 |
| log(消息长度标准差) | 长度是否稳定 |
| log(活跃时间跨度) | 是新用户还是老用户 |
| log(日均频率) | 每天发多少条 |

**举例**：Alice 日均发30条消息，平均每条20字，跨度90天。如果目标平台某用户也是日均30条、每条20字左右，这个维度的相似度就高。但由于这个信号区分度相对弱（很多人活跃度差不多），权重只给了 0.10。

---

## 融合与排序

对每个待对齐的源用户，5个维度分别计算与所有目标用户的余弦相似度，然后加权求和：

```
总分 = 0.25×文本相似 + 0.30×GNN文本相似 + 0.15×时间相似 + 0.20×群组结构相似 + 0.10×活跃度相似
```

按总分降序取 top-K 作为候选列表输出。

---

## 评估指标

用 ground truth（已知的源→目标映射）评估排序质量：

- **hit@1**：正确目标排在第1位的比例。比如100个待对齐用户中，60个的正确答案恰好排第1 → hit@1=0.60
- **hit@K**：正确目标出现在前K个候选中的比例。K=10时，80个用户的正确答案在前10里 → hit@10=0.80
- **MRR**（Mean Reciprocal Rank）：正确答案排名倒数的平均值。如果3个用户的正确答案分别排第1、第3、第5，MRR = (1/1 + 1/3 + 1/5) / 3 ≈ 0.511。MRR 越接近1说明正确答案越靠前
