
## 二、环境配置

### 2.1 Python 版本

推荐 **Python 3.10 / 3.11**。

### 2.2 安装依赖

```bash
pip install -r requirements.txt
```

`requirements.txt` 内容说明：

| 包 | 用途 |
|----|------|
| numpy / pandas / scipy | 基础数值计算 |
| scikit-learn | GNN / Style 特征计算 |
| tqdm | 进度条 |
| torch | Topic 方法的 embedding 推理 |
| sentence-transformers | Topic 方法，默认模型 BAAI/bge-m3（约 570 MB） |
| openai | UserPicture 方法，调用本地 LLM |

#### CPU 环境（无 GPU）安装 torch：
```bash
pip install torch --index-url https://download.pytorch.org/whl/cpu
pip install -r requirements.txt
```

#### GPU 环境（CUDA 12.4）安装 torch：
```bash
pip install torch --index-url https://download.pytorch.org/whl/cu124
pip install -r requirements.txt
```

### 2.3 UserPicture 方法额外要求

UserPicture 方法依赖一个**兼容 OpenAI API 的本地 LLM 服务**（如 vLLM、Ollama 等）。
默认连接地址为 `http://127.0.0.1:6006/v1`，默认模型名 `qwen3`。

可通过环境变量覆盖：
```bash
export OPENAI_BASE_URL=http://127.0.0.1:6006/v1
export OPENAI_MODEL=your_model_name
```

## 三、数据准备

### 3.1 必需的四个 CSV 文件

**① 源/目标平台消息文件**（`source_file` / `target_file`）

| 列名 | 类型 | 说明 |
|------|------|------|
| `account_id` | string | 用户 ID |
| `room_id` | string | 群组 ID |
| `msg` | string | 消息文本 |
| `time` | int | Unix 时间戳（秒） |
| `type` | string | 消息类型，如 `text` / `image` / `video` |

**② 待对齐账号列表**（`need_align_file`）

| 列名 | 类型 | 说明 |
|------|------|------|
| `account_id` | string | 需要在目标平台查找对应账号的源平台用户 ID |

**③ 真值对应表**（`ground_truth_file`）

| 列名 | 类型 | 说明 |
|------|------|------|
| `source_account` | string | 源平台用户 ID |
| `target_account` | string | 对应的目标平台用户 ID |

### 3.2 UserPicture 方法额外需要的画像文件（可选）

如果使用 UserPicture 方法，需要先生成画像 CSV，见第四节。

## 四、UserPicture 画像预生成（仅使用该方法时需要）

> 如果使用 `--disable_user_picture` 跳过此方法，可直接跳到第五节。

### 步骤 1：聚合用户文本

```bash
python user_picture/aggregate_user_text.py \
    data/source_alignment_ground_truth.csv \
    data/source_content.csv

python user_picture/aggregate_user_text.py \
    data/target_alignment_ground_truth.csv \
    data/target_content.csv
```

输出格式（`source_content.csv` / `target_content.csv`）：

| 列名 | 说明 |
|------|------|
| `account_id` | 用户 ID |
| `room_id` | 所在群组 ID（取第一个） |
| `content` | 该用户所有 text 类型消息拼接 |

### 步骤 2：调用 LLM 生成画像

确保本地 LLM 服务已启动，然后运行：

```bash
python user_picture/picture.py \
    data/source_content.csv \
    data/source_profile.csv \
    --model qwen3

python user_picture/picture.py \
    data/target_content.csv \
    data/target_profile.csv \
    --model qwen3
```

输出格式（`source_profile.csv` / `target_profile.csv`）：

| 列名 | 说明 |
|------|------|
| `account_id` | 用户 ID |
| `profile_json` | LLM 生成的结构化画像 JSON |

## 五、运行主程序

### 5.1 完整四路融合（推荐）

```bash
python align_ensemble.py \
    --source_file       data/source_alignment_ground_truth.csv \
    --target_file       data/target_alignment_ground_truth.csv \
    --need_align_file   data/source_need_align_accounts.csv \
    --ground_truth_file data/ground_truth_mapping.csv \
    --source_profile_file data/source_profile.csv \
    --target_profile_file data/target_profile.csv \
    --output_file       results/ensemble_out.csv \
    --k 10
```

### 5.2 不使用 UserPicture（无需 LLM 服务）

```bash
python align_ensemble.py \
    --source_file       data/source_alignment_ground_truth.csv \
    --target_file       data/target_alignment_ground_truth.csv \
    --need_align_file   data/source_need_align_accounts.csv \
    --ground_truth_file data/ground_truth_mapping.csv \
    --output_file       results/ensemble_out.csv \
    --disable_user_picture \
    --k 10
```

### 5.3 只跑 GNN + Style（跳过 Topic 和 UserPicture，速度最快）

```bash
python align_ensemble.py \
    --source_file       data/source_alignment_ground_truth.csv \
    --target_file       data/target_alignment_ground_truth.csv \
    --need_align_file   data/source_need_align_accounts.csv \
    --ground_truth_file data/ground_truth_mapping.csv \
    --output_file       results/ensemble_out.csv \
    --disable_topic \
    --disable_user_picture \
    --k 10
```

### 5.4 所有参数说明

| 参数 | 必填 | 默认值 | 说明 |
|------|------|--------|------|
| `--source_file` | ✓ | — | 源平台消息 CSV |
| `--target_file` | ✓ | — | 目标平台消息 CSV |
| `--need_align_file` | ✓ | — | 待对齐账号列表 CSV |
| `--ground_truth_file` | ✓ | — | 真值对应表 CSV |
| `--output_file` | ✓ | — | 融合结果输出路径 |
| `--k` | | 10 | Top-K 候选数量 |
| `--source_profile_file` | | 空 | 源平台画像 CSV（UserPicture 必需） |
| `--target_profile_file` | | 空 | 目标平台画像 CSV（UserPicture 必需） |
| `--disable_gnn` | | false | 跳过 GNN 方法 |
| `--disable_style` | | false | 跳过 Style 方法 |
| `--disable_user_picture` | | false | 跳过 UserPicture 方法 |
| `--disable_topic` | | false | 跳过 Topic 方法 |
| `--topic_device` | | auto | Topic embedding 设备：`auto` / `cpu` / `cuda` / `cuda:0` |
| `--topic_num_topics` | | 80 | Topic KMeans 聚类数 |

## 七、常见问题

**Q：运行 Topic 方法时提示下载模型很慢？**

第一次运行会自动从 HuggingFace 下载 `BAAI/bge-m3`（约 570 MB）。
可提前手动下载后指定本地路径，或使用镜像源：
```bash
export HF_ENDPOINT=https://hf-mirror.com
```

**Q：UserPicture 方法报错 `Connection refused`？**

本地 LLM 服务未启动，或端口不是 6006。检查服务状态，或通过环境变量修改：
```bash
export OPENAI_BASE_URL=http://127.0.0.1:你的端口/v1
```

**Q：只有少量用户时 Topic 方法效果差？**

KMeans 聚类需要足够的消息量。消息总数少于 1000 条时，建议降低 `--topic_num_topics`（如改为 20），或直接 `--disable_topic`。

**Q：运行速度很慢？**

- GNN + Style 纯 CPU，速度最快（百用户级在 1 分钟内）
- Topic 需要 embedding 推理，建议使用 GPU（`--topic_device cuda`）
- UserPicture 速度取决于 LLM 推理速度，可并行部署多实例
- 快速验证可先只运行 `--disable_topic --disable_user_picture`
